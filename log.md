Сделал папку static (пока пуста) там хочу размещать html страницы, которые будет отдавать сервис:

* GET / (без подпути) - общая страница с названием сервиса, версии и ссылкой на другие страницы
* GET /doc - документация swagger, тут рисовать страницу не нужно
* GET /stat - общая статистика сервиса в виде дашборда в меняющимися в реалтайме значениями
* GET /chat - возможность написать промт и получить на него ответ
* GET /log/core - просмотр "основго лога" (таблицы log, logExtra)
* GET /log/chat - просмот лога промтов (таблицы promt, promtExtra)


1. Все страницы будут использовать единый стиль (рассмотреть создение общего CSS). В любом случае стиль должен быть аккуратный и минималистичный
2. Все страницы будут содержать в себе локально загруженный vue3 (min версия, НЕ CDN! а локально скачанная)
3. Рассмотреть использование какие-нибудь библиотек типа TailWind. Чтобы не использовалось, тоже должно быть локально скачанным

Прочие моменты
1. Если нужно получать данные а реалтайм, то периодическими POST запросами раз, например, в 30 секунд
2. Использовать традиционные HTML страницы с Vue монтированием (Каждая страница - отдельный .html файл, Vue3 подключается в каждой странице и монтируется на #app)
3. Метрики для /stat дашборда продумаем потом отдельно
4. Фильтрацию логов продумаем потом отдельно
5. Все страницы сверху соодержат баннер, содержащий
    * большим шрифтом - одинаковый текст "MENTATOR-LLM_SERVICE, VERSION X.Y.Z"
    * переход на страницу '/'
    * шрифтом поменьше - название страницы (welcome, chat, statistic, log core, log chat)
6. Далее ниже я буду иногда использовать слово "карточка". Это:
    * карточки фиксированной высоты и ширины, которые распологаются последовательно на одной "строке", потом когда места не хватает - переходят на другую "строку" и т.д.
    * при этом если "строка" не последняя, то карточки пропорционально расширяются, чтобы занять свободное на "строке место"
    * на каждой странице и для разных ситуаций фиксированная ширина и высота задается отднльно и не является константой одновроеменно для всех страниц
7. В CSS лучше сразу выделить четыре цвета - b-main1 (темнозеленый), b-main2 (5% от main), b-warn1 (темно оранжевый), b-warn2 (5% от warn), f-main1 (пока белый), f-main2 (пока черный)
8. Если речь идет о выпадающем комбобоксе, текстовом поле и.д. - все они должны быть "повышенной" высоты, что label был вписан "внутрь" рамки этого элемента (в верхнем левом углу). Соответственно другие элементы (типа кнопки) тоже должны быть таеой же высоты, чтобы все элементы на одной строке былы иодной высоты

Надо создать страницу /chat. Делать ее будем в несколько этапов. Для начала просто верстка (без локики, данные - заглушки), указано "по строкам":
1. нижеуказанные элементы:
    * слева, фиксированная ширина - выпадайка с типом промта (system, user)
    * слева, ширина такая, чтобы элементы в строке заняли все место - выпадайка с выбором модели
    * справа, кнопка "format"
    * справа, кнопка "options"
2. шириной на все окно, высота - см. п.4, подсказка "request"
3. нижеуказанные элементы:
    * слева, кнопка "load"
    * справа, внопка "send"
4. горизонтальный сплиттер, делит по умолчанию страницу (не считая баннер, на две части по 50%). Т.е. мемо выше и мемо ниже будут занимать по вертикали столько места, сколько им будет оставаться доступным
5. мемо шириной на все окно, высота - см. п.3, подсказка "response"
6. нижеуказанные элементы:
    * справа кнопка "to clipboard all"
    * справа кнопка "to clipboard payload"
    * справа кнопка "to file all"
    * справа кнопка "to file payload"
