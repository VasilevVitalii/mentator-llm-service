Реализация страницы /log/chat
Много аналогий со страницей /log/core.
Бъется также на три похожие части.
Верхняя часть точно такая же, только в DOWNLOAD спрашивается "YYYYMMDD HH" (т.е. еще и час).
Нижняя часть точно такая же, только нет чекбокса "ADDITIONAL DATA ..."
Стоится на основе таблиц promt и promtExtra.
Каждое событие - 3 строки
1. Строка-заглоовок состоит из:
[вроемя как в /log/core] [поле code из promt] duration promt XXX; duration queue YYY, где XXX и YYY - соответствующие поля из promt, показывать время в формате "как на карточке "server info" страницы "/stat". Только еще милисекунды добавить.

Далее (такое же форматирование как для logExtra, в том числе отступ):

2. если есть данные в promtExtra.request (напомню, что не всегда вообще есть строки для promtExtra), то показывается
```
request
<первые 500 символов>
```
2. если есть данные в promtExtra.response (напомню, что не всегда вообще есть строки для promtExtra), то показывается
```
response
<первые 500 символов>
```

Клик на request и response уберает ограничение "первые 500 символов" и показывает все данные, повторный клик возвращает обратно "первые 500 символов".

На "to clipboart current" и "to file current" видимость "500 символов или нет" - не влияет, там всегда целиком данные выгружаются (с учетом ERROR ONLY)

Ошибками сыитаюся все, где Code<>200
